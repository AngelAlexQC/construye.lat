## Design Context

### Users
Developers worldwide — all levels, all stacks. From senior engineers at startups to solo founders building MVPs. They want an AI coding agent that's fast, capable, and doesn't treat them like beginners. They live in the terminal. They've tried Claude Code, Codex, Gemini CLI. They're looking for something better.

### Brand Personality
**Relentless. Universal. Precise.**

- **Relentless**: Never stops until the job is done. The name is a command: "construye" = "build." It's an imperative. An instruction. A mindset.
- **Universal**: Not regional. The .lat domain is an accent, not a border. Like how "sudo" is universal despite being English. The tool speaks the developer's language — literally and metaphorically.
- **Precise**: No wasted tokens, no wasted keystrokes, no wasted time. Every interaction is intentional. Close to AGI means close to understanding what you actually want.

### Aesthetic Direction
**Industrial minimalism with warmth.** Think: a perfectly organized workshop. Tools where you need them, no clutter, warm lighting. Not cold/sterile tech aesthetic. Not playful/toy-like. Not dark-mode-neon-glow AI slop.

- **Terminal-native**: This is a CLI. It should feel like the best possible terminal experience, not a web app crammed into a terminal.
- **Information density done right**: Power users want density. But density ≠ chaos. It's about rhythm, hierarchy, and letting the content breathe at exactly the right moments.
- **Distinctive without trying**: The identity comes from how it behaves, not from decorative flourishes. Speed IS the brand. Intelligence IS the design.

### Anti-References
- NOT ChatGPT (corporate, generic, lowest-common-denominator)
- NOT Claude Code (good but we're building something original, not a tribute)
- NOT toy CLIs with excessive emojis and rainbow gradients
- NOT cold/sterile "hacker" aesthetic with green-on-black matrix vibes
- NOT generic "AI startup" with cyan gradients and glassmorphism

### Design Principles

1. **Speed is felt, not claimed**: Don't show a "fast" badge. Make every interaction feel instant. Streaming that starts before you finish reading the prompt. Tool calls that appear to execute in parallel. The UI should feel like it's always one step ahead.

2. **Content is the interface**: The most important thing on screen is always the agent's output or the user's input. Everything else (status, metrics, tool calls) serves the content, never competes with it.

3. **Progressive density**: Start clean, reveal complexity through interaction. A new user sees simplicity. A power user discovers depth. No modes, no settings panels — the interface adapts.

4. **Rhythm over decoration**: Visual hierarchy comes from spacing, timing, and typography weight — not from colored borders, icons, or emoji. The rhythm of appearing text, the pause before a tool call, the subtle shift when thinking ends and responding begins.

5. **The terminal is a canvas, not a constraint**: Ink + React means we have flexbox, components, state management. Use the terminal's constraints as creative fuel: monospace becomes a grid system, limited colors become a disciplined palette, the cursor becomes the only animation you need.
